# 공공직군 행정업무 슈퍼앱 PRD

## 1. 문서 목적

본 문서는 공공직군 행정업무 슈퍼앱의 초기 기능 요구사항을 정의한다. 개발 착수 전 제품 범위, 사용자, 핵심 기능, 우선순위, 비기능 요구사항, 향후 검토 항목을 명확히 하는 것을 목적으로 한다.

## 2. 제품 개요

공공직군 행정업무 슈퍼앱은 조직 내 반복 행정업무를 하나의 웹 애플리케이션에서 처리하도록 지원하는 내부 업무 도구다. 초기 버전은 팀 일정 공유, 엑셀 업무 자동화, 민원 대응 보조 챗봇, 행정 뉴스 수집 기능을 제공한다.

## 3. 대상 사용자

| 사용자 | 주요 니즈 |
| --- | --- |
| 팀장/관리자 | 팀원의 휴가, 근무, 출장 현황을 빠르게 파악하고 업무 공백을 관리 |
| 일반 팀원 | 본인 일정을 등록하고 다른 팀원 일정을 확인 |
| 행정 담당자 | 반복적인 엑셀 분리/병합 작업을 자동화 |
| 민원 담당자 | 매뉴얼 기반으로 일관된 민원 대응 방식과 응대 스크립트 확보 |
| 정책/홍보 담당자 | 공공 행정 관련 뉴스를 매일 아침 확인 |

## 4. 핵심 문제

- 팀 일정이 메신저, 구두 보고, 별도 문서에 흩어져 있어 실시간 공유가 어렵다.
- 엑셀 데이터를 특정 기준으로 나누거나 합치는 반복 작업에 시간이 많이 든다.
- 민원 대응 시 담당자별 답변 품질과 표현이 달라질 수 있다.
- 공공 행정 관련 뉴스를 수동으로 검색해야 하며 누락 가능성이 있다.

## 5. 기능 요구사항

### 5.1 팀원 스케쥴 관리

#### 설명
팀원들의 휴가, 근무, 출장 등 일정을 캘린더에서 공유하고 관리한다.

#### 주요 기능
- 월간/주간/일간 캘린더 조회
- 일정 유형 등록: 휴가, 근무, 출장, 교육, 기타
- 일정 제목, 기간, 담당자, 메모 입력
- 팀원별/일정 유형별 필터링
- 일정 중복 또는 장기 부재 확인
- 오늘 일정 및 다가오는 일정 요약

#### 우선순위
- MVP: 일정 등록, 조회, 수정, 삭제, 유형 필터
- 후속: 승인 워크플로우, 알림, 조직도 연동

#### 수용 기준
- 사용자는 팀원별 일정을 캘린더에서 확인할 수 있다.
- 사용자는 본인 또는 권한이 있는 팀원의 일정을 등록/수정/삭제할 수 있다.
- 일정 유형별 색상이 구분되어야 한다.

### 5.2 엑셀 업무 자동화

#### 설명
사용자가 업로드한 엑셀 파일을 특정 컬럼 기준으로 나누거나 여러 엑셀 파일을 하나로 합친다.

#### 주요 기능
- `.xlsx` 파일 업로드
- 특정 컬럼의 값 기준으로 파일 분리
- 여러 엑셀 파일 병합
- 병합 시 헤더 기준 일치 여부 검증
- 처리 결과 파일 다운로드
- 처리 이력 확인

#### 우선순위
- MVP: 단일 파일 컬럼 기준 분리, 다중 파일 병합, 결과 다운로드
- 후속: 템플릿 저장, 컬럼 매핑, 대용량 비동기 처리

#### 수용 기준
- 사용자는 컬럼명을 선택하고 해당 값 기준으로 엑셀을 여러 파일로 분리할 수 있다.
- 사용자는 여러 엑셀 파일을 하나의 파일로 병합할 수 있다.
- 파일 형식 오류나 컬럼 누락 시 명확한 에러 메시지가 제공되어야 한다.

### 5.3 민원 대응 챗봇

#### 설명
첨부된 민원 매뉴얼을 바탕으로 민원 대응 방식, 답변 초안, 응대 스크립트를 작성하는 챗봇을 제공한다.

#### 주요 기능
- 민원 매뉴얼 파일 업로드 및 관리
- 민원 내용 입력
- 관련 매뉴얼 근거 검색
- 답변 초안 생성
- 전화/방문/서면 대응 스크립트 생성
- 답변 생성 시 참고 근거 표시
- 부적절한 확정 답변 방지를 위한 검토 안내 표시

#### 우선순위
- MVP: 매뉴얼 기반 질의응답, 답변 초안 생성, 근거 표시
- 후속: 문서 버전 관리, 부서별 매뉴얼 분리, 승인된 답변 템플릿 관리

#### 수용 기준
- 챗봇 답변은 업로드된 매뉴얼의 내용을 우선 근거로 사용해야 한다.
- 근거가 부족한 경우 임의 확정 답변 대신 담당자 검토 필요를 안내해야 한다.
- 생성된 답변은 민원 담당자가 수정 가능한 초안 형태로 제공되어야 한다.

### 5.4 뉴스 기사 수집

#### 설명
공공 행정과 관련된 뉴스 기사를 매일 아침 수집하고 요약 목록을 제공한다.

#### 주요 기능
- 키워드 기반 뉴스 수집
- 매일 아침 정해진 시간 자동 수집
- 기사 제목, 출처, 발행일, 링크 저장
- 중복 기사 제거
- 주요 기사 요약
- 키워드별 분류

#### 우선순위
- MVP: 수동 수집, 키워드 기반 저장, 목록 조회
- 후속: 예약 수집, 요약, 이메일/메신저 알림

#### 수용 기준
- 사용자는 수집된 기사 목록을 날짜별로 확인할 수 있다.
- 동일 URL 또는 동일 제목의 중복 기사는 제거되어야 한다.
- 수집 실패 시 원인과 재시도 가능 여부가 기록되어야 한다.

## 6. 비기능 요구사항

| 항목 | 요구사항 |
| --- | --- |
| 보안 | 내부 업무 데이터, 민원 문서, 업로드 파일은 외부 공개되지 않아야 한다. |
| 개인정보 | 민원 데이터에는 개인정보가 포함될 수 있으므로 저장, 조회, 로그 처리 시 최소 수집 원칙을 적용한다. |
| 감사 추적 | 일정 변경, 파일 처리, 챗봇 답변 생성, 뉴스 수집 실행 이력을 기록한다. |
| 성능 | 일반적인 엑셀 파일 처리와 캘린더 조회는 3초 이내 응답을 목표로 한다. |
| 안정성 | 파일 처리 실패, 뉴스 수집 실패, 챗봇 근거 부족 상황을 사용자에게 명확히 안내한다. |
| 확장성 | 초기 DB는 SQLite를 사용하되, 향후 PostgreSQL 등으로 이전 가능하도록 저장소 계층을 분리한다. |

## 9. 리스크 및 검토 항목

- 민원 매뉴얼 기반 챗봇은 법적/행정적 책임이 발생할 수 있으므로 최종 답변은 담당자 검토를 전제로 해야 한다.
- 뉴스 수집은 언론사 약관, 포털 정책, 저작권 이슈를 검토해야 한다.
- 엑셀 파일에는 개인정보나 민감정보가 포함될 수 있으므로 파일 보관 기간과 삭제 정책이 필요하다.
- SQLite는 초기 개발과 소규모 운영에는 적합하지만 동시 쓰기 부하가 커지면 DB 전환이 필요하다.
